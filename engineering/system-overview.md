# System Overview

## Product Thesis

The product is a dental loyalty and patient retention platform, not a generic bonus
widget. It should help clinics increase repeat visits and make return behavior visible,
manageable, and economically safe.

## Primary Business Outcome

Improve repeat visit behavior while keeping clinic operations simple and loyalty
economics controlled.

## Core Actors

- `patient`
- `front_desk`
- `clinic_manager`
- `owner`
- `doctor`
- `platform_support`

## Value Exchange

### Patient

- sees wallet value clearly
- can inspect visit, payment, and loyalty history
- receives reminders and return nudges at the right time
- feels trust because balance changes are explainable

### Front Desk

- sees the patient context fast
- can confirm payment-related actions confidently
- can apply allowed redemptions without policy guesswork
- can escalate exceptions without breaking the flow

### Clinic Manager

- controls clinic-level loyalty policy
- reviews manual interventions
- monitors operational discipline and campaign execution

### Owner

- sees repeat-visit ROI
- understands bonus liability and redemption behavior
- compares clinic or branch performance
- gets a SaaS product tied to measurable operational value

### Doctor

- sees only the patient information needed for treatment context
- does not manage loyalty economics directly

## Product Backbone

The platform centers on this loop:

`patient -> visit -> payment -> loyalty ledger -> communication -> return`

The loop matters more than any single bonus rule.

## MVP Backbone

The first commercial MVP should include:

- authentication and role-based access
- patient profile
- visits
- payments
- loyalty wallet
- loyalty ledger
- manual adjustment with reason capture
- audit visibility
- patient cabinet
- basic reminders
- owner-facing KPI visibility

## Explicit Exclusions Before Proof

- complex gamification
- uncontrolled reward catalogs
- AI scoring
- heavy white-label custom work
- marketplace-style expansion
- advanced partner ecosystems

## Canonical Terms

Use these terms consistently:

- retention platform
- confirmed payment
- loyalty wallet
- loyalty ledger
- tenant
- audit trail
- manual fallback
- owner-visible ROI
